为什么切片可以实现对已有的数组的引用，当切片为空时，又能够自动开辟内存空间存储需要添加的值

在 Go 中，切片是对底层数组的一个抽象。切片本身是一个结构体，包含三个字段：指向底层数组的指针、切片的长度和切片的容量。通过这种方式，切片可以引用底层数组的一部分，并且可以动态调整其长度和容量。

### 切片的内部结构

切片的内部结构如下：

type SliceHeader struct {

    Data uintptr

    Len  int

    Cap  int

}

- **Data**：指向底层数组的指针。
- **Len**：切片的长度，即切片中实际包含的元素数量。
- **Cap**：切片的容量，即从切片的起始位置到底层数组末尾的元素数量。

### 切片的内存管理

1. **引用底层数组**：当创建一个切片时，切片会引用一个底层数组。切片的 `Data` 字段指向这个数组的起始位置，`Len` 字段表示切片的长度，`Cap` 字段表示切片的容量。
2. **动态扩展**：当使用 `append` 函数向切片添加元素时，如果切片的长度超过了容量，Go 运行时会自动分配一个更大的底层数组，并将原来的数据复制到新的数组中。然后，切片的 `Data` 字段会更新为指向新的数组，`Len` 和 `Cap` 字段也会相应更新。



这里有一点小提示：由于切片的灵活性，大部分情况下用数组其实都是在用切片，因为数组Array只支持固定长度且在初始化时就必须确定长度
